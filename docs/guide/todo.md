## 建议下一步

建议按以下优先级继续推进：

### P0 前端 schema 协议最终收口

- 补齐更多布局节点与真实字段节点的区分规则
- 明确更多字段约束到数据库字段定义和运行时校验规则的映射
- 固定前端字段别名、默认值、显示态、校验消息等协议

### P1 发布 / 迁移增强

- 明确列表、详情、筛选、排序对展示值字段的支持边界
